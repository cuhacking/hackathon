### Configure Git

Your username and email will be used to identify you in the Git commit history.
    ```sh title="Terminal"
    # Set your username
git config --global user.name "Your Name"
    ```

    ```sh title="Terminal"
    # Set your email
git config --global user.email "your_email@example.com"
    ```

    Git's default branch is `master`, let's change it to `main` to match the convention used by GitHub.
    ```sh title="Terminal"
    # Set the default branch to main
git config --global init.defaultBranch main
    ```

<Callout type="warn" title="Git and GitHub are different things">
     Git is the tool that manages code history, while GitHub is a cloud platform for storing and sharing repositories.
     - Like what Microsoft Word is to Onedrive, or Google Docs is to Google Drive.
</Callout>

### Check Your Setup

Confirm that your setup is correct by running these commands.

```sh title="Terminal"
# Check Homebrew installation
brew --version

# Check Git version
git --version

# Confirm GitHub CLI installation
gh --version

# Verify global Git config
git config --list
```

If everything is working correctly, you should see the relevant version numbers and configuration details.

- The term `directory` means the same thing as folder.
- The `~` symbol represents the home directory in both Linux and Unix (MacOS is UNIX).
- Everything in Linux is a file.
- Every command begins with a program name.
- You can learn more about the program using the `man` program.

Try the following:
```sh title="Terminal"
man cd
```
Press `q` to quit. You can put almost any program after the `man` command to learn more about it. You can even do `man man`.

### Turn the current directory into a Git repository
Now we'll start using `git`.

```sh title="Terminal"
git init .
```
A _repository_ is a _directory_ that is version controlled by Git.


   ```sh title="Terminal"
   git add <your_file_name>
   ```

   ```sh title="Terminal"
   git commit -m "Initial commit"
   ```
